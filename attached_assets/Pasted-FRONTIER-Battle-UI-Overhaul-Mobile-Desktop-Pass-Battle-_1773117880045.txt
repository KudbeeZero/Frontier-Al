ATTACKMODAL FIXES:

1. Dialog sizing — replace the outer DialogContent className with:
"w-full max-w-lg mx-auto max-h-[85vh] overflow-y-auto backdrop-blur-md"
Remove the sm:max-w-lg prefix so it takes full width on mobile with consistent padding.

2. Commander cards — change the card dimensions from w-[72px] h-24 to w-16 h-20 and reduce the horizontal scroll container padding. On mobile the "No Cmd" + commander row should scroll horizontally without clipping.

3. Troop counter layout — the ChevronDown/ChevronUp buttons with the font-mono number between them should use a min-w-[2.5rem] on the number span so it doesn't shift layout when going from 1 to 10 troops.

4. Slider labels — each slider row currently uses flex items-center justify-between. Add a min-h-[1.75rem] to each label row so they don't collapse on small screens.

5. Power display grid — the grid-cols-2 gap-4 p-4 block showing Your Power / Defender Power: increase the font size of the labels from text-xs to text-sm and add a min-h-[5rem] to the grid container so it doesn't collapse on narrow screens.

6. Cost summary row — the "Total Cost" row at the bottom uses flex items-center justify-between. Wrap the cost badges in a flex-wrap gap-2 container so they stack on very narrow screens instead of overflowing.

7. Warning block — the yellow warning block above the footer uses space-y-1. Add p-3 rounded-md border to the outer div if not already present. Ensure each warning line has flex items-start gap-1.5 (not items-center) so long messages wrap cleanly.

8. DialogFooter — add flex-col sm:flex-row gap-2 to the footer so Cancel and Deploy stack vertically on mobile and sit side by side on desktop. Both buttons should be w-full sm:w-auto.
