# FRONTIER ŌĆö Fix MemStorage ŌåÆ DbStorage (Replit)

## Context

You are working inside the **Frontier-Al** Replit project. This is a production MMO strategy game running on Replit with a PostgreSQL database.

**DO NOT** refactor, rename, restructure, or touch anything outside the files listed below. Make only the three precise changes described.

-----

## Problem

The server always boots with `MemStorage` (in-memory, ephemeral storage) regardless of whether `DATABASE_URL` is set in the environment. This means **all game state is wiped on every restart**. `DbStorage` is fully implemented and working ŌĆö it is simply never instantiated. The fix is surgical.

-----

## Fix 1 ŌĆö `server/storage.ts`

Find the **very last line** of the file:

```typescript
export const storage: IStorage = new MemStorage();
```

Replace it with:

```typescript
export const storage: IStorage = process.env.DATABASE_URL
  ? new DbStorage()
  : (() => {
      console.warn("[STORAGE] No DATABASE_URL found ŌĆö falling back to MemStorage. All game state is ephemeral and will reset on restart.");
      return new MemStorage();
    })();
```

-----

## Fix 2 ŌĆö `server/index.ts`

Find this line inside the async IIFE:

```typescript
await registerRoutes(httpServer, app);
```

Add the following block **directly above** it:

```typescript
if (process.env.NODE_ENV === "production" && !process.env.DATABASE_URL) {
  console.error("[FATAL] DATABASE_URL is required in production. Provision a PostgreSQL database in Replit and ensure the secret is set. Exiting.");
  process.exit(1);
}
```

-----

## Fix 3 ŌĆö Delete `railway.toml`

Delete the file `railway.toml` from the project root. This project runs on Replit, not Railway. The file is unused and misleading.

-----
