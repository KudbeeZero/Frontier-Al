FRONTIER — NFT Opt-In + Auto-Deliver UX
Pass: NFT-Deliver-UX

Context:
The Claim NFT button already exists in LandSheet.tsx and the
handleDeliverNft handler already exists in GameLayout.tsx.
The problem is UX — when a player gets the "not_opted_in" error,
they have no in-game way to opt into the specific plot NFT ASA.
They have to go into Pera wallet manually and find the ASA by ID.

This pass adds:
1. An "Opt In & Claim" flow — when deliver returns not_opted_in,
   prompt the player to sign an opt-in transaction for the specific
   ASA, then automatically retry the deliver call.
2. A clearer NFT status display in LandSheet showing the ASA ID
   as a tappable Pera deep-link so the player can manually opt-in
   if they prefer.

DO NOT TOUCH: server files, storage.ts, db-schema.ts, wsServer.ts,
or any client file not listed below.

Files to modify:
- client/src/components/game/GameLayout.tsx
- client/src/components/game/LandSheet.tsx

Replace with:
  const handleDeliverNft = async () => {
    if (!ownedSelectedPlotId || !wallet.address || !nftInfo) return;
    setIsDeliveringNft(true);

    const attemptDeliver = async (): Promise<void> => {
      const res = await fetch(`/api/nft/deliver/${ownedSelectedPlotId}`, {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({ address: wallet.address }),
      });
      const data = await res.json();

      if (!res.ok) {
        toast({ title: "NFT Delivery Failed", description: data.error || "Unknown error", variant: "destructive" });
        return;
      }

      if (data.success) {
        toast({ title: "NFT Delivered! 🎉", description: `Plot #${ownedSelectedPlotId} NFT is now in your wallet.` });
        refetchNft();
        return;
      }

      if (data.reason === "not_opted_in") {
        // Wallet hasn't opted into this ASA yet — prompt them to opt in,
        // then automatically retry the deliver
        const optedIn = await signOptInToPlotNft(nftInfo.assetId);
        if (optedIn) {
          // Small delay to let the opt-in transaction propagate
          await new Promise(resolve => setTimeout(resolve, 2000));
          // Retry deliver once
          const retryRes = await fetch(`/api/nft/deliver/${ownedSelectedPlotId}`, {
            method: "POST",
            headers: { "Content-Type": "application/json" },
            body: JSON.stringify({ address: wallet.address }),
          });
          const retryData = await retryRes.json();
          if (retryData.success) {
            toast({ title: "NFT Delivered! 🎉", description: `Plot #${ownedSelectedPlotId} NFT is now in your wallet.` });
            refetchNft();
          } else {
            toast({ title: "Claim Failed", description: "Opt-in confirmed but delivery failed. Try claiming again in a moment.", variant: "destructive" });
          }
        }
        return;
      }

      if (data.reason === "not_in_custody") {
        toast({ title: "Already In Your Wallet", description: data.message || "NFT is already delivered." });
        refetchNft();
        return;
      }

      toast({ title: "Claim Issue", description: data.message || "Unexpected state — try again.", variant: "destructive" });
    };

    try {
      await attemptDeliver();
    } catch (err) {
      toast({ title: "NFT Delivery Error", description: err instanceof Error ? err.message : "Unexpected error", variant: "destructive" });
    } finally {
      setIsDeliveringNft(false);
    }
  };
