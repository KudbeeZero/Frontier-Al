# FRONTIER — Fix Missing Table & Build Allowlist

## Context

You are working inside the **Frontier-Al** Replit project. This is a production MMO strategy game.

**DO NOT** refactor, rename, restructure, or touch anything outside the two files listed below. Make only the exact changes described.

-----

## Problem 1 — `mint_idempotency` table is never auto-created

`DbStorage._doInitialize()` in `server/storage.ts` manually creates safety-net tables on boot using `CREATE TABLE IF NOT EXISTS`. The `mint_idempotency` table is **missing from this list** but is actively used by the land purchase flow in `server/routes.ts`. If this table doesn’t exist, any land purchase attempt will crash the server with a relation-not-found error.

## Problem 2 — `algosdk` and wallet packages missing from build allowlist

In `script/build.ts`, the `allowlist` array controls which packages get **bundled** into the production binary. Any package NOT on the list is externalized — meaning it must exist as a `node_modules` entry at runtime. On Replit Autoscale deployments, externalized packages that aren’t explicitly installed at runtime will cause **module-not-found crashes** on any blockchain call. `algosdk`, `@perawallet/connect`, and `lute-connect` are all missing from the allowlist.

-----

## Fix 1 — `server/storage.ts`

Find this block inside `_doInitialize()` (around line 1542):

```typescript
    await this.db.execute(sql`
      CREATE TABLE IF NOT EXISTS orbital_events (
        id               VARCHAR(36) PRIMARY KEY,
        type             VARCHAR(30) NOT NULL,
        cosmetic         BOOLEAN NOT NULL DEFAULT FALSE,
        start_at         BIGINT NOT NULL,
        end_at           BIGINT NOT NULL,
        seed             INT NOT NULL DEFAULT 0,
        intensity        REAL NOT NULL DEFAULT 0.5,
        trajectory       JSONB NOT NULL DEFAULT '{}',
        target_parcel_id VARCHAR(36),
        effects          JSONB NOT NULL DEFAULT '[]',
        resolved         BOOLEAN NOT NULL DEFAULT FALSE
      )
    `);
```

Add the following block **directly after** it (after the closing backtick and semicolon):

```typescript
    await this.db.execute(sql`
      CREATE TABLE IF NOT EXISTS mint_idempotency (
        key         TEXT PRIMARY KEY,
        status      VARCHAR(10) NOT NULL DEFAULT 'pending',
        asset_id    BIGINT,
        tx_id       TEXT,
        created_at  BIGINT NOT NULL,
        updated_at  BIGINT NOT NULL
      )
    `);
```
