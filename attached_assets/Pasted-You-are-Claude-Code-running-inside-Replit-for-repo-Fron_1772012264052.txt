T003 — FIX FlatMap texture alignment (FlatMap.tsx)
========================================================
Goal: When dragging globe left/right, terrain/night/cloud textures visibly scroll with plots (not fixed).
Also: fix the “banding / half-blank” issue near poles by CLAMPING latitude vertical offset.

FILE
- `client/src/components/game/FlatMap.tsx`

TARGET
- The draw block where textures are drawn like:
  `ctx.drawImage(img, cx - R, cy - R, R * 2, R * 2)`

REPLACE LOGIC
- Compute offsets each frame:
  - W = R*2
  - H = R*2
  - xOff = -(centerLng / 360) * W
  - wrap xOff so it stays bounded
- yOff should be SUBTLE and CLAMPED to avoid the “half-blank” band:
  - yOff = (centerLat / 180) * (R * 0.35)
  - clamp to [-R*0.35, +R*0.35]

- Draw each layer 3 times horizontally to avoid dateline tearing:
  - at x0, x0+W, x0-W

- Clouds get slight parallax:
  - extraX = W * 0.03 (or 0.02), so clouds drift slightly differently.

IMPLEMENTATION SNIPPET (use exactly this pattern inside the draw scope)
(Place this near where you currently draw the layers)

```ts
const lng = cameraRef.current.centerLng;
const lat = cameraRef.current.centerLat;

const W = R * 2;
const H = R * 2;

// Horizontal texture scroll (wrap to prevent drift)
let xOff = -(lng / 360) * W;
xOff = ((xOff % W) + W) % W;
xOff = xOff - W;

// Vertical scroll is only an approximation — keep subtle + clamp to prevent banding
let yOff = (lat / 180) * (R * 0.35);
yOff = Math.max(-R * 0.35, Math.min(R * 0.35, yOff));

const drawWrapped = (img: CanvasImageSource, extraX = 0, extraY = 0) => {
  const x0 = cx - R + xOff + extraX;
  const y0 = cy - R + yOff + extraY;

  ctx.drawImage(img, x0, y0, W, H);
  ctx.drawImage(img, x0 + W, y0, W, H);
  ctx.drawImage(img, x0 - W, y0, W, H);
};
