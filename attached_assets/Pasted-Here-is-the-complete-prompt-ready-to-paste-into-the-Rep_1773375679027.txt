## FRONTIER — Owned Tile Ambient Breathe + Color Fix — Pass TileBreath

### Context

Two problems are being fixed in this single pass:

1. Owned parcels appear black because `getPlotColor` returns `UNOWNED_DIM` when the
   `players` array hasn't resolved yet, causing `isOwned` (derived from color sum) to
   be false and the tile to render at scale=0.

2. Owned tiles are completely static. We want all owned tiles to slowly "breathe" —
   a gentle, slow brightness oscillation that makes the globe feel alive — with each
   tile out of phase with its neighbours so the effect is organic rather than
   synchronized.

Performance contract:
- `latLngToVec3` must NOT be called inside `useFrame`. Positions are pre-cached into
  two `useMemo` arrays (fill layer and border layer) computed once on mount.
- `instanceMatrix.needsUpdate` is only set when a matrix actually changed (scale/pos).
  The breathe effect is colour-only, so it never dirties the matrix buffer.
- `instanceColor.needsUpdate` is set per frame only when colour changed.
- Breathe tiles are processed in rolling buckets of 200 per frame, spreading CPU cost
  across ~100 frames. At 60fps this means the full globe completes one breathe pass
  every ~1.7 seconds — invisible to the player, zero frame budget impact.
- `isOwned` must be derived from `parcel?.ownerId != null`, not from a colour sum.
  Colour-sum ownership was the root cause of the black tile bug.

DO NOT TOUCH: any server files, `routes.ts`, `wsServer.ts`, `GameLayout.tsx`,
`GlobeEventOverlays.tsx`, hover logic, selected tile logic, battle tile logic,
or the `useEffect` static initialiser.
