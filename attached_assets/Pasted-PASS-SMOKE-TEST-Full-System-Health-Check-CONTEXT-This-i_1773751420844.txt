PASS: SMOKE-TEST — Full System Health Check

CONTEXT:
This is a non-destructive read-only smoke test pass for FRONTIER. Do NOT modify any game logic,
balance constants, DB schema, battle engine, AI engine, trade system, or WebSocket code.
The sole purpose is to run a structured diagnostic across every major subsystem and report
pass/fail with exact error output. No fixes in this pass — report only.

---

STEP 1 — TypeScript Compile Check

Run:
  npx tsc --noEmit

Capture all errors. Report count and each error with file + line. If zero errors, mark ✅ PASS.

---

STEP 2 — Environment Variable Audit

Check that the following env vars are present (non-empty) in the Replit Secrets or .env:

Required:
  - DATABASE_URL
  - UPSTASH_REDIS_REST_URL
  - UPSTASH_REDIS_REST_TOKEN
  - SESSION_SECRET
  - ALGORAND_NETWORK (expected: "testnet")
  - PUBLIC_BASE_URL (must NOT fall back to req.get("host"))

Optional but flag if missing:
  - AI_ENABLED
  - FORCE_NEW_ASA

For each: ✅ PRESENT / ❌ MISSING / ⚠️ SUSPICIOUS VALUE

---

STEP 3 — Database Connectivity

Run a raw Drizzle/Neon connectivity check:
  SELECT 1 AS ping;

Then verify these tables exist in the DB:
  - players
  - parcels
  - battles
  - trade_orders
  - trade_fills
  - orbital_events
  - season_state (if exists)

For each: ✅ EXISTS / ❌ MISSING

Also check: does the `battles` table have a `source_parcel_id` column?
  → ✅ PRESENT (Prompt A migration applied) / ❌ MISSING (Prompt A not yet run)

---

STEP 4 — Redis Connectivity

Using the Upstash REST client (server/services/redis.ts), run:
  - PING check (getRedis() returns non-null)
  - Write a test key: SET frontier:smoke_test "ok" EX 60
  - Read it back: GET frontier:smoke_test
  - Delete it: DEL frontier:smoke_test

Report: ✅ PASS / ❌ FAIL with error

---

For each: ✅ 200 / ⚠️ 401/403 (auth-gated, acceptable) / ❌ 500 (report full error body)

STEP 6 — Battle Engine Unit Smoke

Import resolveBattle from server/engine/battle/resolve.ts and run this fixture:

  const result = resolveBattle({
    attackerId: "smoke-attacker",
    defenderId: "smoke-defender",
    attackerTroops: 100,
    defenderTroops: 80,
    attackerIron: 50,
    defenderIron: 40,
    attackerFuel: 30,
    defenderFuel: 20,
    biome: "plains",
    defenderImprovementLevel: 2,
    orbitalHazardActive: false,
    randomSeed: 42,
    attackerMorale: 1.0,
    defenderMorale: 1.0,
  });

Check:
  - result is not null/undefined ✅
  - result.winner is "attacker" or "defender" ✅
  - result.log is a non-empty array ✅
  - result is deterministic: run it twice with seed 42, both outputs identical ✅
